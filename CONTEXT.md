# MYhao's Portfolio

个人生涯展示博客，包含游戏测评、作品展示、学习笔记、职业历程等板块。

## Language

**最近更新内容 (Recent Updates)**:
首页的混合内容栏目，按发布时间倒序展示所有公开内容（游戏测评 + 作品集 + 学习笔记），以分页形式排列。
_避免_: 精选内容、Featured

**文章详情页 (Article Detail Page)**:
游戏测评、作品集、学习笔记三种内容类型的详情页面，均以 MDX 渲染正文，右侧配置目录。
_避免_: 详情、帖子页

**目录 (Table of Contents)**:
从文章 Markdown 正文中自动提取 h2 标题生成的侧边导航，桌面端 sticky 显示，移动端折叠到顶部。
_避免_: 大纲、TOC

**头像 (Avatar)**:
个人形象图片，同时显示在 Header 导航栏左侧和关于我页面，点击可跳转到 /about。
_避免_: Logo、头像图标

**后台音乐控制 (Background Music Control)**:
当浏览器标签页切换到后台时自动暂停 BGM，回到前台后不自动恢复，需用户手动点击播放按钮。
_避免_: 自动暂停、静音控制

## Flagged ambiguities

- **"项目"**: 用户使用该词时需确认指的是作品集（portfolio）中的项目，还是混合内容中的条目。当前约定：在首页「最近更新内容」语境下，"项目"指所有内容类型（测评+作品+笔记）的统称；在 `/portfolio` 语境下仅指作品集。

## Example dialogue

> **Dev**: "首页的最近更新内容是只显示项目的，还是所有内容？"
>
> **Domain expert**: "所有内容混在一起，按时间排序。游戏测评、作品、学习笔记都要出现。"
>
> **Dev**: "那分页是传统页码还是加载更多？"
>
> **Domain expert**: "传统页码，URL 里带 page 参数，方便分享。"
>
> **Dev**: "文章详情页的目录是固定侧边栏还是只有顶部？"
>
> **Domain expert**: "桌面端固定右侧，移动端缩小了就收到文章顶部，变成一个可展开的下拉。BGM 切到后台自动停，回来不自动放。"
